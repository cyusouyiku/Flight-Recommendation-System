一、项目总览
================================================================

项目名称：机票推荐系统（AI版）
后端框架：Spring Boot 3.4.1 + Spring Security + MyBatis
前端框架：Vue 3 + TypeScript + Vite
数据库：H2（开发内存库）/ 支持切换 MySQL、PostgreSQL（生产）
核心特色：集成 DeepSeek AI，对推荐结果进行智能重排，实现用户反馈闭环优化

-- airport（机场表）
CREATE TABLE airport (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    code VARCHAR(3) NOT NULL UNIQUE,       -- 机场代码 (PEK, PVG等)
    name VARCHAR(255),                     -- 机场名称
    city VARCHAR(255),                     -- 城市
    country VARCHAR(255),                  -- 国家
    timezone VARCHAR(64)                   -- 时区
);

-- airline（航空公司表）
CREATE TABLE airline (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    code VARCHAR(2) NOT NULL UNIQUE,       -- 航司代码 (CA, MU等)
    name VARCHAR(255)                      -- 航司名称
);

-- route（航线表）
CREATE TABLE route (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    departure_airport_code VARCHAR(3),
    arrival_airport_code VARCHAR(3),
    departure_airport_name VARCHAR(255),
    arrival_airport_name VARCHAR(255)
);

-- flight（航班主表）
CREATE TABLE flight (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    departure_time TIMESTAMP,
    arrival_time TIMESTAMP,
    duration_minutes INT,
    effective_date DATE,
    flight_number VARCHAR(32),
    airline_code VARCHAR(8),
    cabin_class VARCHAR(32),               -- Y/E/C/F
    aircraft_type VARCHAR(32),
    stops INT,
    is_direct BOOLEAN,
    departure_airport VARCHAR(255),
    arrival_airport VARCHAR(255),
    departure_airport_code VARCHAR(3),
    arrival_airport_code VARCHAR(3),
    price INT,
    original_price INT,
    available_seats INT,
    status VARCHAR(32),
    created_at TIMESTAMP,
    updated_at TIMESTAMP
);

-- app_user（用户表）
CREATE TABLE app_user (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    username VARCHAR(64),
    password VARCHAR(255),                 -- BCrypt加密
    email VARCHAR(255),
    phone VARCHAR(32),
    created_at TIMESTAMP,
    updated_at TIMESTAMP
);

-- booking（预订记录表）
CREATE TABLE booking (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    user_id BIGINT,
    flight_number VARCHAR(32),
    cabin_class VARCHAR(32),
    passenger_name VARCHAR(255),
    status VARCHAR(32),
    amount INT,
    created_at TIMESTAMP,
    updated_at TIMESTAMP
);

-- flight_price（价格历史表）
CREATE TABLE flight_price (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    flight_number VARCHAR(32),
    departure_airport_code VARCHAR(3),
    arrival_airport_code VARCHAR(3),
    cabin_class VARCHAR(32),
    price INT,
    record_at TIMESTAMP
);

-- user_flight_history（用户行为历史表）
CREATE TABLE user_flight_history (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    user_id BIGINT,
    departure_airport_code VARCHAR(3),
    arrival_airport_code VARCHAR(3),
    behavior_type VARCHAR(32),             -- SEARCH/CLICK/BOOK
    flight_number VARCHAR(32),
    created_at TIMESTAMP
);

-- price_alert（低价提醒表）
CREATE TABLE price_alert (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    user_id BIGINT NOT NULL,
    departure_airport_code VARCHAR(3) NOT NULL,
    arrival_airport_code VARCHAR(3) NOT NULL,
    target_price INT NOT NULL,
    cabin_class VARCHAR(32) DEFAULT 'Y',
    notified BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMP,
    updated_at TIMESTAMP
);

-- user_feedback（用户反馈表）
CREATE TABLE user_feedback (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    user_id BIGINT,
    flight_id BIGINT,
    feedback_type VARCHAR(32) NOT NULL,    -- LIKE/DISLIKE/NEUTRAL
    feedback_text TEXT,
    ai_analysis TEXT,
    confidence_score DECIMAL(3,2),         -- 0-1
    metadata TEXT,                         -- JSON格式元数据
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES app_user(id) ON DELETE SET NULL,
    FOREIGN KEY (flight_id) REFERENCES flight(id) ON DELETE SET NULL
);
CREATE INDEX idx_user_feedback_user_id ON user_feedback(user_id);
CREATE INDEX idx_user_feedback_feedback_type ON user_feedback(feedback_type);
CREATE INDEX idx_user_feedback_created_at ON user_feedback(created_at);

5.1 推荐算法（四层策略）
--------------------------------------------------------------

第1层：热门推荐
  统计全局搜索频次最高的 Top5 航线，取每条航线最便宜的航班

第2层：常搜航线
  获取当前登录用户搜索频次 Top3 的航线，取未来14天内的航班

第3层：你可能喜欢
  获取用户常搜 Top2 航线，推荐用户从未搜索过的相似航线航班

第4层：随机推荐（兜底）
  未登录用户或无历史数据时使用

        ↓ 以上结果全部汇入 AI 重排 ↓

AI 重排逻辑：
  1. 合并所有层推荐的候选航班
  2. 获取用户历史反馈（LIKE/DISLIKE/NEUTRAL）
  3. 构建 FlightRerankPrompt（含8个排序因素：价格、时间、航司、转机、舒适度等）
  4. 调用 DeepSeek API，返回每个航班的评分 JSON（flight_id, score 0-1）
  5. 过滤掉分数 < 0.7 的航班
  6. 按分数重新排序，返回优化后的推荐列表


5.2 AI 服务（四大功能）
--------------------------------------------------------------

功能                          说明
--------------------------------------------------------------
rerankRecommendations         用户历史反馈 + 航班信息 → AI 打分 → 重新排序
generatePersonalizedRecs      根据用户偏好（时间/价格/舱位）→ AI 生成个性化推荐
analyzeUserFeedback           对用户反馈做情感分析，提取关键因素和改进建议
analyzeRecommendationPerf     对推荐系统整体效果做 SWOT 分析

AI 调用链路：
  PromptBuilder（构建 Prompt）
      → AiClient（调用 DeepSeek API，使用 OpenAI Java SDK）
      → AiResponseParser（解析 JSON 响应）
      → AiServiceImpl（整合业务逻辑）

AiClient 核心配置：
  model: deepseek-chat
  temperature: 0.3（低随机性，结果更稳定）
  max-tokens: 1000
  timeout: 30s
  base-url: https://api.deepseek.com/v1

PromptBuilder 构建的 Prompt 类型：
  1. FlightRerankPrompt     - 航班重排，返回 { sorted_flights: [ { flight_id, score, reason } ] }
  2. FeedbackAnalysisPrompt - 反馈分析，返回 { sentiment, sentiment_score, key_factors, confidence }
  3. RecommendationPerformancePrompt - 效果分析，返回 SWOT 分析 JSON

AiResponseParser 解析逻辑：
  - 使用 Jackson ObjectMapper 解析 AI 返回的 JSON 字符串
  - 容错处理：解析失败时返回默认结果，不抛出异常

5.4 其他服务
--------------------------------------------------------------

PriceAlertService：
  用户设置价格阈值，定时任务每分钟扫描 price_alert 表
  当航班价格低于阈值时，调用 NotificationService 发送邮件通知
  notified 字段标记已通知，避免重复发送

PriceTrendService：
  查询 flight_price 表中某航线的历史价格记录
  按时间排序返回，前端绘制价格趋势折线图

UserBehaviorService：
  每次用户搜索/点击/预订时，异步写入 user_flight_history 表
  供 RecommendationService 的第2、3层推荐算法使用

NotificationService：
  封装 Spring Mail，发送 HTML 格式邮件
  需在 application.yml 配置 SMTP 信息

9.1 Vite 代理配置（vite.config.ts）
--------------------------------------------------------------
前端端口：3000
代理规则：/api → http://localhost:8080
作用：开发时将所有 /api 请求转发到 Spring Boot 后端，避免跨域问题

9.2 API 客户端（utils/api.ts）
--------------------------------------------------------------
基于 Axios 封装，baseURL = '/api'，超时 10000ms

请求拦截器：
  自动从 localStorage 读取 token，添加 Authorization: Bearer xxx 请求头

响应拦截器：
  统一处理 Result<T> 格式响应（code=0 为成功）
  HTTP 401 → 清除 token，跳转登录页
  HTTP 403 → 提示无权限
  HTTP 500 → 提示服务器错误

9.3 认证状态管理（stores/auth.ts）
--------------------------------------------------------------
使用 Pinia 管理用户登录状态
持久化到 localStorage（token, user 信息）
提供 login(), logout(), isAuthenticated 等方法
路由守卫使用 auth store 判断是否需要重定向到登录页

十一、项目亮点与注意事项
================================================================

亮点：
  1. AI 重排是闭环设计
     用户提交反馈 → AI 分析情感和原因 → 下次推荐时 AI 利用反馈数据重新排序
     形成持续优化的推荐飞轮

  2. 推荐算法分层清晰
     冷启动用全局热门，有历史用个人常搜，进一步挖掘用相似航线
     AI 作为最终优化层，而不是唯一依赖

  3. AI 降级保护
     AI 不可用（API Key 无效、网络超时等）时，系统自动降级返回原始推荐结果
     不抛出异常，不影响主流程

  4. 开箱即用
     使用 H2 内存数据库 + Flyway 自动迁移，无需手动建表和导入数据
     git clone 后直接 mvn spring-boot:run 即可启动

  5. 接口文档完善
     集成 SpringDoc OpenAPI，Swagger UI 提供所有接口的在线文档和测试能力

注意事项：
  1. AI 功能需要配置 DeepSeek API Key
     通过环境变量 DEEPSEEK_API_KEY 或直接修改 application.yml 配置
     不配置时 AI 功能静默降级，不会报错

  2. H2 是内存数据库，重启后数据全部清空
     Flyway 会在每次启动时重新执行种子数据脚本，测试数据会自动恢复
     生产环境需切换到 MySQL/PostgreSQL

  3. 邮件通知（低价提醒）需配置 SMTP
     不配置不影响其他所有功能，仅 price_alert 邮件发送不生效

  4. Redis 默认禁用
     如需启用缓存，移除 autoconfigure.exclude 配置并添加 Redis 连接信息

  5. 定时任务
     系统每分钟扫描一次 price_alert 表，检查是否有需要发送提醒的记录
     可在日志中看到 scheduling-1 线程的 SQL 执行记录
